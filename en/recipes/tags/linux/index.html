Настройка параметров последовательного порта в Linux

December 7, 2025 · 3 min read
dmn
maintainer

Установка и проверка параметров последовательного порта в Linux на примере ttyS1.

Просмотр текущих параметров​

stty -F /dev/ttyS1 -a

Пример вывода:​

speed 115200 baud; line = 0;
-brkint -icrnl -imaxbel
opost -onlcr
cs8 -parenb -cstopb

Расшифровка параметров​

Скорость и базовые настройки:​

  • speed 115200 baud - текущая скорость передачи
  • cs8 - 8 бит данных
  • -parenb - без бита четности (N)
  • -cstopb - 1 стоп-бит (если cstopb → 2 стоп-бита)

Управляющие сигналы:​

  • -crtscts - отключить аппаратный контроль потока RTS/CTS
  • -ixon -ixoff - отключить программный контроль потока XON/XOFF

Установка параметров порта​

Типичная конфигурация:​

stty -F /dev/ttyS1 115200 cs8 -cstopb -parenb -ixon -ixoff -crtscts

Разбор команды:​

  • 115200 - скорость 115200 бод
  • cs8 - 8 бит данных
  • -cstopb - 1 стоп-бит
  • -parenb - без четности
  • -ixon -ixoff - без программного контроля потока
  • -crtscts - без аппаратного контроля потока

Популярные конфигурации​

Modbus RTU (9600 8N1):​

stty -F /dev/ttyS1 9600 cs8 -cstopb -parenb -ixon -ixoff -crtscts

Промышленные датчики (19200 8E1):​

stty -F /dev/ttyS1 19200 cs8 -cstopb parenb parodd -ixon -ixoff -crtscts

GPS модули (4800 8N1):​

stty -F /dev/ttyS1 4800 cs8 -cstopb -parenb -ixon -ixoff -crtscts

Тестирование порта​

Отправка данных:​

echo "test message" > /dev/ttyS1

Чтение данных:​

cat /dev/ttyS1

Интерактивная работа:​

# В одном терминале - чтение
cat /dev/ttyS1

# В другом терминале - отправка  
echo "hello" > /dev/ttyS1

Специальные режимы​

Raw режим (без обработки):​

stty -F /dev/ttyS1 raw

Восстановление в cooked режим:​

stty -F /dev/ttyS1 cooked

Минимальные настройки:​

stty -F /dev/ttyS1 115200 raw -echo

Сохранение настроек​

Сохранение текущих настроек:​

stty -F /dev/ttyS1 > /tmp/ttyS1_settings.txt

Восстановление настроек:​

stty -F /dev/ttyS1 $(cat /tmp/ttyS1_settings.txt)

Отладка проблем​

Проверка доступности устройства:​

ls -la /dev/ttyS1

Проверка прав доступа:​

# Добавление пользователя в группу dialout
sudo usermod -a -G dialout $USER

Мониторинг активности:​

# Просмотр статистики порта
cat /proc/tty/driver/serial

Автоматизация настроек​

Скрипт инициализации:​

#!/bin/bash
# setup_serial.sh
DEVICE="/dev/ttyS1"
BAUDRATE="115200"

stty -F $DEVICE $BAUDRATE cs8 -cstopb -parenb -ixon -ixoff -crtscts
echo "Port $DEVICE configured: $BAUDRATE 8N1"

Systemd сервис:​

[Unit]
Description=Serial Port Setup
After=multi-user.target

[Service]
Type=oneshot
ExecStart=/usr/local/bin/setup_serial.sh
RemainAfterExit=yes

[Install]
WantedBy=multi-user.target

Правильная настройка параметров последовательного порта критически важна для стабильной работы с промышленным оборудованием и датчиками.

Просмотр опций компиляции ядра Linux

October 21, 2025 · 2 min read
dmn
maintainer

Как посмотреть с какими опциями было скомпилировано текущее ядро Linux.

Основная команда​

zcat /proc/config.gz

Эта команда выводит полную конфигурацию ядра, с которой оно было скомпилировано.

Фильтрация по интересующим опциям​

Все настройки последовательных портов:​

zcat /proc/config.gz | grep -E 'CONFIG_SERIAL_8250_NR_UARTS|CONFIG_SERIAL_8250|CONFIG_SERIAL_'

Настройки GPIO:​

zcat /proc/config.gz | grep CONFIG_GPIO

Настройки SPI:​

zcat /proc/config.gz | grep CONFIG_SPI

Настройки I2C:​

zcat /proc/config.gz | grep CONFIG_I2C

Настройки USB:​

zcat /proc/config.gz | grep CONFIG_USB

Поиск конкретных опций​

Поиск по названию модуля:​

zcat /proc/config.gz | grep -i modbus
zcat /proc/config.gz | grep -i zigbee  
zcat /proc/config.gz | grep -i bluetooth

Поиск драйверов:​

zcat /proc/config.gz | grep CONFIG_DRIVER

Полезные примеры​

Проверка поддержки UART:​

zcat /proc/config.gz | grep CONFIG_SERIAL_8250_NR_UARTS
# Покажет максимальное количество поддерживаемых UART портов

Проверка поддержки CAN:​

zcat /proc/config.gz | grep CONFIG_CAN

Проверка поддержки Modbus:​

zcat /proc/config.gz | grep CONFIG_MODBUS

Альтернативные способы​

Через /boot:​

# Если конфиг сохранен в /boot
cat /boot/config-$(uname -r)

Через модули:​

# Список загруженных модулей
lsmod

# Информация о конкретном модуле
modinfo <module_name>

Анализ результатов​

Значения опций:

  • =y - встроено в ядро
  • =m - скомпилировано как модуль
  • # CONFIG_XXX is not set - опция отключена

Пример вывода:​

CONFIG_SERIAL_8250=y
CONFIG_SERIAL_8250_CONSOLE=y
CONFIG_SERIAL_8250_NR_UARTS=4
CONFIG_SERIAL_8250_RUNTIME_UARTS=4
CONFIG_SERIAL_8250_DW=y

Практическое применение​

Отладка проблем:​

  • Проверить включена ли поддержка нужного интерфейса
  • Убедиться что драйвер скомпилирован
  • Найти ограничения (например, количество UART)

Планирование:​

  • Определить возможности текущего ядра
  • Понять нужна ли пересборка для новых функций

Эта информация особенно полезна при работе с embedded системами и отладке проблем совместимости оборудования.

Диагностика SD-NAND карт в Linux - получение регистров

August 31, 2025 · 2 min read
dmn
maintainer

При работе с SD-NAND картами в ядрах Linux 6.x иногда возникают проблемы с инициализацией некоторых карт, например Zetta. Для диагностики полезно уметь получать данные регистров карт.

Получение данных регистров SD-NAND карт​

Основные регистры для диагностики:​

# Card Identification Register
cat /sys/bus/mmc/devices/mmc0\:1388/cid

# Card Specific Data Register  
cat /sys/bus/mmc/devices/mmc0\:1388/csd

# Operation Conditions Register
cat /sys/bus/mmc/devices/mmc0\:1388/ocr

# Relative Card Address
cat /sys/bus/mmc/devices/mmc0\:1388/rca

# SD Card Configuration Register
cat /sys/bus/mmc/devices/mmc0\:1388/scr

Пример вывода для разных карт:​

Карта 1:

root@napi-rk3308b-s:~# cat /sys/bus/mmc/devices/mmc0\:1388/cid
ba2345534134474206071c06fe016945

root@napi-rk3308b-s:~# cat /sys/bus/mmc/devices/mmc0\:1388/csd
400e00325b5900001cdf7f800a4000d7

root@napi-rk3308b-s:~# cat /sys/bus/mmc/devices/mmc0\:1388/ocr
0x00300000

root@napi-rk3308b-s:~# cat /sys/bus/mmc/devices/mmc0\:1388/rca
0x1388

root@napi-rk3308b-s:~# cat /sys/bus/mmc/devices/mmc0\:1388/scr
0235000e00000000

Карта 2:

root@napi-nightstarfish:~# cat /sys/bus/mmc/devices/mmc0\:1388/cid
e700115a6574746101b135ee4a719523

Расшифровка регистров​

CID (Card Identification):​

  • Содержит уникальную информацию о производителе
  • Серийный номер карты
  • Дату производства

CSD (Card Specific Data):​

  • Параметры доступа к данным
  • Размер карты
  • Скоростные характеристики

OCR (Operation Conditions):​

  • Поддерживаемые уровни напряжения
  • Готовность карты к работе

Поиск проблем​

Определение адреса устройства:​

# Найти все MMC устройства
ls /sys/bus/mmc/devices/

# Проверить все доступные регистры
ls -la /sys/bus/mmc/devices/mmc0:*/

Проверка совместимости:​

# Информация о хост-контроллере
cat /proc/driver/mmc0

# Логи инициализации карты
dmesg | grep mmc

Практическое применение​

Эта информация особенно важна при:

  • Диагностике проблем совместимости с новыми ядрами
  • Выборе подходящих SD-NAND карт для проекта
  • Отладке проблем инициализации в embedded системах

Сравнение регистров разных карт помогает выявить различия в их характеристиках и понять причины проблем совместимости с конкретными версиями ядра Linux.

Чтение и программирование EEPROM преобразователя FT232

August 15, 2025 · 2 min read
dmn
maintainer

Полное руководство по чтению и программированию внутренней прошивки преобразователя RTU-USB FT232 в Linux.

Установка программного обеспечения​

sudo apt install libftdi1-2 libftdi1-dev ftdi-eeprom

Чтение EEPROM​

1. Создание конфигурационного файла​

touch c.txt  # создаем пустой файл

# vendor/product смотрим в lsusb
echo 'vendor_id=0x0403
product_id=0x6001' > c.txt

2. Чтение содержимого EEPROM​

ftdi_eeprom --read-eeprom c.txt --verbose

Пример успешного вывода:​

FTDI eeprom generator v0.17
(c) Intra2net AG and the libftdi developers <opensource@intra2net.com>
FTDI read eeprom: 0
EEPROM size: 128
VID:     0x0403
PID:     0x6001
Release: 0x0000
Bus Powered:  90 mA USB Remote Wake Up
Manufacturer: FTDI
Product:      FT232R USB UART
Serial:       A10M5ZU3
Checksum      : 17cb
Internal EEPROM
Oscillator: Internal

PNP: 1

C0 Function: TXLED
C1 Function: RXLED

C4 Function: SLEEP
Warning: Not writing eeprom, you must supply a valid filename
FTDI close: 0

Зачем это нужно?​

🙋‍♀️ Основные причины:

  • Проверить жив ли FT232 - диагностика работоспособности чипа
  • Проверить назначение линий CBUS (C0-C4) - которые могут быть переопределены для специальных функций

Полезные факты​

👍 Диагностические команды:

Проверка USB конфигурации:​

lsusb -v  # должно быть состояние bConfigurationValue = 1

Проверка состояния PWREN#:​

  • После инициализации драйвером должен быть в НУЛЕ (0)

Понимание SLEEP#:​

  • SLEEP# = 1 - USB-шина в нормальном рабочем состоянии
  • SLEEP# = 0 - устройство в режиме suspend (спящий режим)

Сохранение и программирование​

Сохранение сырого EEPROM:​

sudo ftdi_eeprom --read-eeprom текущий.conf --device ft232 --small-eeprom --eeprom-filename дамп.bin

Создание кастомного конфига (ftdi.conf):​

vendor_id=0x0403
product_id=0x6001
manufacturer="MyCompany"
product="My FT232R"
serial="FT123456"

# CBUS configuration:
cbus0=1   # Можно оставить как есть или назначить PWREN
cbus1=1   # PWREN, к примеру
cbus2=0   # TXDEN
cbus3=2   # RXLED (имитация RXDEN через LED#)
cbus4=4   # TXRXLED

Прошивка новой конфигурации:​

sudo ftdi_eeprom --flash-eeprom ftdi.conf

Возможные проблемы​

При ошибке "device not found" попробуйте:

  1. Проверить подключение устройства: lsusb
  2. Проверить права доступа к USB устройствам
  3. Убедиться что драйвер не занял устройство: sudo rmmod ftdi_sio usbserial

Этот инструмент позволяет полностью контролировать конфигурацию FT232 и адаптировать его под специфические требования проекта.
