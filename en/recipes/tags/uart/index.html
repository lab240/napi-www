Настройка параметров последовательного порта в Linux

December 7, 2025 · 3 min read
dmn
maintainer

Установка и проверка параметров последовательного порта в Linux на примере ttyS1.

Просмотр текущих параметров​

stty -F /dev/ttyS1 -a

Пример вывода:​

speed 115200 baud; line = 0;
-brkint -icrnl -imaxbel
opost -onlcr
cs8 -parenb -cstopb

Расшифровка параметров​

Скорость и базовые настройки:​

  • speed 115200 baud - текущая скорость передачи
  • cs8 - 8 бит данных
  • -parenb - без бита четности (N)
  • -cstopb - 1 стоп-бит (если cstopb → 2 стоп-бита)

Управляющие сигналы:​

  • -crtscts - отключить аппаратный контроль потока RTS/CTS
  • -ixon -ixoff - отключить программный контроль потока XON/XOFF

Установка параметров порта​

Типичная конфигурация:​

stty -F /dev/ttyS1 115200 cs8 -cstopb -parenb -ixon -ixoff -crtscts

Разбор команды:​

  • 115200 - скорость 115200 бод
  • cs8 - 8 бит данных
  • -cstopb - 1 стоп-бит
  • -parenb - без четности
  • -ixon -ixoff - без программного контроля потока
  • -crtscts - без аппаратного контроля потока

Популярные конфигурации​

Modbus RTU (9600 8N1):​

stty -F /dev/ttyS1 9600 cs8 -cstopb -parenb -ixon -ixoff -crtscts

Промышленные датчики (19200 8E1):​

stty -F /dev/ttyS1 19200 cs8 -cstopb parenb parodd -ixon -ixoff -crtscts

GPS модули (4800 8N1):​

stty -F /dev/ttyS1 4800 cs8 -cstopb -parenb -ixon -ixoff -crtscts

Тестирование порта​

Отправка данных:​

echo "test message" > /dev/ttyS1

Чтение данных:​

cat /dev/ttyS1

Интерактивная работа:​

# В одном терминале - чтение
cat /dev/ttyS1

# В другом терминале - отправка  
echo "hello" > /dev/ttyS1

Специальные режимы​

Raw режим (без обработки):​

stty -F /dev/ttyS1 raw

Восстановление в cooked режим:​

stty -F /dev/ttyS1 cooked

Минимальные настройки:​

stty -F /dev/ttyS1 115200 raw -echo

Сохранение настроек​

Сохранение текущих настроек:​

stty -F /dev/ttyS1 > /tmp/ttyS1_settings.txt

Восстановление настроек:​

stty -F /dev/ttyS1 $(cat /tmp/ttyS1_settings.txt)

Отладка проблем​

Проверка доступности устройства:​

ls -la /dev/ttyS1

Проверка прав доступа:​

# Добавление пользователя в группу dialout
sudo usermod -a -G dialout $USER

Мониторинг активности:​

# Просмотр статистики порта
cat /proc/tty/driver/serial

Автоматизация настроек​

Скрипт инициализации:​

#!/bin/bash
# setup_serial.sh
DEVICE="/dev/ttyS1"
BAUDRATE="115200"

stty -F $DEVICE $BAUDRATE cs8 -cstopb -parenb -ixon -ixoff -crtscts
echo "Port $DEVICE configured: $BAUDRATE 8N1"

Systemd сервис:​

[Unit]
Description=Serial Port Setup
After=multi-user.target

[Service]
Type=oneshot
ExecStart=/usr/local/bin/setup_serial.sh
RemainAfterExit=yes

[Install]
WantedBy=multi-user.target

Правильная настройка параметров последовательного порта критически важна для стабильной работы с промышленным оборудованием и датчиками.

Overlay для RS485 на UART1 с GPIO управлением направления

November 29, 2025 · 2 min read
dmn
maintainer

Создание overlay для настройки RS485 на UART1 (NAPI-C) с контролем направления передачи через GPIO2_B4.

Overlay файл uart1-de.dts​

/dts-v1/;
/plugin/;

/ {
  compatible = "rockchip,rk3308";

  fragment@0 {
    target = <&uart1>;
    __overlay__ {
      status = "okay";

      pinctrl-names = "default";
      pinctrl-0 = <&uart1_xfer>;

      linux,rs485-enabled-at-boot-time;
      rs485-rts-active-high;
      rs485-rts-delay = <0 0>;
      rts-gpios = <&gpio2 12 0>;
    };
  };
};

Разбор параметров​

Основные настройки:​

  • status = "okay" - включает UART1
  • pinctrl-0 = <&uart1_xfer> - настройки пинов для TX/RX

RS485 специфичные параметры:​

  • linux,rs485-enabled-at-boot-time - включает RS485 при загрузке
  • rs485-rts-active-high - RTS активен высоким уровнем
  • rs485-rts-delay = <0 0> - задержки переключения [before, after] в микросекундах
  • rts-gpios = <&gpio2 12 0> - GPIO для управления направлением (DE pin)

Расчет GPIO​

Для GPIO2_B4:

  • Банк: 2
  • Группа: B (8-15)
  • Пин: 4
  • Итого: 8 + 4 = 12

Установка overlay​

Компиляция и установка:​

# Компиляция
dtc -@ -I dts -O dtb -o uart1-de.dtbo uart1-de.dts

# Копирование в системную папку
sudo cp uart1-de.dtbo /boot/dtb/overlay/

# Добавление в armbianEnv.txt
echo "user_overlays=uart1-de" >> /boot/armbianEnv.txt

# Перезагрузка
reboot

Автоматический способ (Armbian):​

armbian-add-overlay uart1-de.dts

Проверка работы​

Проверка устройства:​

ls -la /dev/ttyS1

Проверка RS485 параметров:​

# Через device tree
ls -la /proc/device-tree/serial@ff0b0000/
cat /proc/device-tree/serial@ff0b0000/status

# Должны присутствовать файлы:
# linux,rs485-enabled-at-boot-time
# rs485-rts-active-high
# rs485-rts-delay
# rts-gpios

Проверка GPIO:​

gpioinfo -c gpiochip2 | grep "line  12"

Настройка порта​

Установка параметров:​

stty -F /dev/ttyS1 9600 cs8 -cstopb -parenb -ixon -ixoff -crtscts

Проверка настроек:​

stty -F /dev/ttyS1 -a

Тестирование RS485​

Отправка данных:​

echo "test message" > /dev/ttyS1

Мониторинг GPIO:​

# Наблюдение за переключением DE pin
watch -n 0.1 "gpioget -c gpiochip2 12"

Логирование трафика:​

# Отправка с мониторингом
stdbuf -oL cat /dev/ttyS1 | tee rs485.log &
echo "test" > /dev/ttyS1

Альтернативные GPIO​

Если GPIO2_B4 занят, можно использовать:

rts-gpios = <&gpio2 11 0>;  // GPIO2_B3
rts-gpios = <&gpio2 13 0>;  // GPIO2_B5
rts-gpios = <&gpio3 27 0>;  // GPIO3_B3

Отладка проблем​

Проверка конфликтов пинов:​

cat /sys/kernel/debug/pinctrl/pinctrl-rockchip-pinctrl/pinmux-pins | grep gpio2

Логи ядра:​

dmesg | grep uart
dmesg | grep rs485

Данный overlay обеспечивает корректную работу RS485 интерфейса с автоматическим управлением направлением передачи через выделенный GPIO пин.

Просмотр опций компиляции ядра Linux

October 21, 2025 · 2 min read
dmn
maintainer

Как посмотреть с какими опциями было скомпилировано текущее ядро Linux.

Основная команда​

zcat /proc/config.gz

Эта команда выводит полную конфигурацию ядра, с которой оно было скомпилировано.

Фильтрация по интересующим опциям​

Все настройки последовательных портов:​

zcat /proc/config.gz | grep -E 'CONFIG_SERIAL_8250_NR_UARTS|CONFIG_SERIAL_8250|CONFIG_SERIAL_'

Настройки GPIO:​

zcat /proc/config.gz | grep CONFIG_GPIO

Настройки SPI:​

zcat /proc/config.gz | grep CONFIG_SPI

Настройки I2C:​

zcat /proc/config.gz | grep CONFIG_I2C

Настройки USB:​

zcat /proc/config.gz | grep CONFIG_USB

Поиск конкретных опций​

Поиск по названию модуля:​

zcat /proc/config.gz | grep -i modbus
zcat /proc/config.gz | grep -i zigbee  
zcat /proc/config.gz | grep -i bluetooth

Поиск драйверов:​

zcat /proc/config.gz | grep CONFIG_DRIVER

Полезные примеры​

Проверка поддержки UART:​

zcat /proc/config.gz | grep CONFIG_SERIAL_8250_NR_UARTS
# Покажет максимальное количество поддерживаемых UART портов

Проверка поддержки CAN:​

zcat /proc/config.gz | grep CONFIG_CAN

Проверка поддержки Modbus:​

zcat /proc/config.gz | grep CONFIG_MODBUS

Альтернативные способы​

Через /boot:​

# Если конфиг сохранен в /boot
cat /boot/config-$(uname -r)

Через модули:​

# Список загруженных модулей
lsmod

# Информация о конкретном модуле
modinfo <module_name>

Анализ результатов​

Значения опций:

  • =y - встроено в ядро
  • =m - скомпилировано как модуль
  • # CONFIG_XXX is not set - опция отключена

Пример вывода:​

CONFIG_SERIAL_8250=y
CONFIG_SERIAL_8250_CONSOLE=y
CONFIG_SERIAL_8250_NR_UARTS=4
CONFIG_SERIAL_8250_RUNTIME_UARTS=4
CONFIG_SERIAL_8250_DW=y

Практическое применение​

Отладка проблем:​

  • Проверить включена ли поддержка нужного интерфейса
  • Убедиться что драйвер скомпилирован
  • Найти ограничения (например, количество UART)

Планирование:​

  • Определить возможности текущего ядра
  • Понять нужна ли пересборка для новых функций

Эта информация особенно полезна при работе с embedded системами и отладке проблем совместимости оборудования.

Чтение и программирование EEPROM преобразователя FT232

August 15, 2025 · 2 min read
dmn
maintainer

Полное руководство по чтению и программированию внутренней прошивки преобразователя RTU-USB FT232 в Linux.

Установка программного обеспечения​

sudo apt install libftdi1-2 libftdi1-dev ftdi-eeprom

Чтение EEPROM​

1. Создание конфигурационного файла​

touch c.txt  # создаем пустой файл

# vendor/product смотрим в lsusb
echo 'vendor_id=0x0403
product_id=0x6001' > c.txt

2. Чтение содержимого EEPROM​

ftdi_eeprom --read-eeprom c.txt --verbose

Пример успешного вывода:​

FTDI eeprom generator v0.17
(c) Intra2net AG and the libftdi developers <opensource@intra2net.com>
FTDI read eeprom: 0
EEPROM size: 128
VID:     0x0403
PID:     0x6001
Release: 0x0000

Manufacturer: FTDI
Product:      FT232R USB UART
Serial:       A10M5ZU3
Checksum      : 17cb
Internal EEPROM
Oscillator: Internal
Enable Remote Wake Up
PNP: 1
Channel A has Mode UART VCP
C0 Function: TXLED
C1 Function: RXLED
C2 Function: TXDEN
C3 Function: PWREN
C4 Function: SLEEP
Warning: Not writing eeprom, you must supply a valid filename
FTDI close: 0

Зачем это нужно?​

🙋‍♀️ Основные причины:

  • Проверить жив ли FT232 - диагностика работоспособности чипа
  • Проверить назначение линий CBUS (C0-C4) - которые могут быть переопределены для специальных функций

Полезные факты​

👍 Диагностические команды:

Проверка USB конфигурации:​

lsusb -v  # должно быть состояние bConfigurationValue = 1

Проверка состояния PWREN#:​

  • После инициализации драйвером должен быть в НУЛЕ (0)

Понимание SLEEP#:​

  • SLEEP# = 1 - USB-шина в нормальном рабочем состоянии
  • SLEEP# = 0 - устройство в режиме suspend (спящий режим)

Сохранение и программирование​

Сохранение сырого EEPROM:​

sudo ftdi_eeprom --read-eeprom текущий.conf --device ft232 --small-eeprom --eeprom-filename дамп.bin

Создание кастомного конфига (ftdi.conf):​

vendor_id=0x0403
product_id=0x6001
manufacturer="MyCompany"
product="My FT232R"
serial="FT123456"

# CBUS configuration:
cbus0=1   # Можно оставить как есть или назначить PWREN
cbus1=1   # PWREN, к примеру
cbus2=0   # TXDEN
cbus3=2   # RXLED (имитация RXDEN через LED#)
cbus4=4   # TXRXLED

Прошивка новой конфигурации:​

sudo ftdi_eeprom --flash-eeprom ftdi.conf

Возможные проблемы​

При ошибке "device not found" попробуйте:

  1. Проверить подключение устройства: lsusb
  2. Проверить права доступа к USB устройствам
  3. Убедиться что драйвер не занял устройство: sudo rmmod ftdi_sio usbserial

Этот инструмент позволяет полностью контролировать конфигурацию FT232 и адаптировать его под специфические требования проекта.

Конфигурация UART на модуле CM4 в Токосборщике

August 10, 2025 · One min read
dmn
maintainer

В Токосборщике на модуле CM4 UART порты имеют специальное назначение и требуют правильной настройки оверлеев.

Назначение UART портов​

✔️ UART3 - внешний датчик Modbus
✔️ UART9 - модуль расширений (Zigbee)
✔️ UART7 - встроенный датчик тока

Конфигурация оверлеев​

Для корректной работы в Debian необходимо подключить оверлеи с uart7,9 в файле /boot/orangepiEnv.txt:

verbosity=1
bootlogo=false
extraargs=cma=128M
overlay_prefix=rk356x
overlays=uart7-m2 uart9-m2
rootdev=UUID=a0f8ca89-7eb7-4a1e-947a-2341637b4782
rootfstype=ext4
console=serial

Проверка конфигурации​

Просмотр текущей конфигурации:

cat /boot/orangepiEnv.txt

Проверка активации​

После перезагрузки проверьте наличие устройств:

ls -la /dev/ttyS*

Должны появиться соответствующие устройства:

  • /dev/ttyS3 - для внешнего Modbus датчика
  • /dev/ttyS7 - для встроенного датчика тока
  • /dev/ttyS9 - для модуля расширений Zigbee

Эта конфигурация обеспечивает корректную работу всех интерфейсов в системе Токосборщика на базе CM4.

Подключение UART4 в NapiLinux

July 16, 2025 · One min read
dmn
maintainer

Для подключения UART4 в NapiLinux необходимо выполнить несколько простых шагов.

Пошаговая инструкция​

1. Создание папки для пользовательских оверлеев​

mkdir -p /boot/overlay-user

2. Размещение файла оверлея​

Положите в созданную папку файл rk3308-uart4.dtbo:

# Убедитесь, что файл находится в нужном месте
ls -la /boot/overlay-user/rk3308-uart4.dtbo

3. Настройка загрузки​

Добавьте строчку в файл /boot/uEnv.txt:

echo "user_overlays=rk3308-uart4" >> /boot/uEnv.txt

4. Перезагрузка системы​

reboot

5. Проверка результата​

После перезагрузки должно появиться устройство /dev/ttyS4:

ls -la /dev/ttyS4

Проверка работоспособности​

Для проверки работы UART4 можно использовать:

# Проверка доступности устройства
stat /dev/ttyS4

# Отправка тестовых данных
echo "test" > /dev/ttyS4

Данный метод позволяет быстро активировать дополнительный UART интерфейс в системе NapiLinux без необходимости пересборки ядра.
